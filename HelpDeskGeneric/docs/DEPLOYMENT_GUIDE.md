## 3. ソリューションのエクスポート

### 3.1 Power Apps ポータルからエクスポート

1. **https://make.powerapps.com/** にサインイン
2. 左メニューの「**ソリューション**」をクリック
3. 「**汎用ヘルプデスク（デバッグ）**」（HelpDeskGeneric）を選択
4. 上部の「**エクスポート**」をクリック
5. エクスポートの種類を選択：
   - **マネージド**: 本番環境向け（カスタマイズ不可）
   - **アンマネージド**: ステージング/検証環境向け（カスタマイズ可）
6. 「**エクスポート**」をクリックし、ZIP ファイルをダウンロード

### 3.2 PAC CLI でエクスポート（オプション）

```powershell
# 認証
pac auth create --name SourceEnv --environment {source-environment-id}

# マネージドとしてエクスポート
pac solution export --name HelpDeskGeneric --path ./HelpDeskGeneric_managed.zip --managed true

# アンマネージドとしてエクスポート
pac solution export --name HelpDeskGeneric --path ./HelpDeskGeneric_unmanaged.zip --managed false
```

## 4. ソリューションのインポート

### 4.1 Power Apps ポータルからインポート

1. 導入先環境の **https://make.powerapps.com/** にサインイン
2. **正しい環境を選択**（右上の環境セレクター）
3. 左メニューの「**ソリューション**」をクリック
4. 上部の「**インポート**」→「**ソリューションをインポート**」
5. エクスポートした ZIP ファイルを選択
6. 「**次へ**」をクリック

#### 接続の設定

インポート中に **接続の設定** を求められます。以下の接続を事前に作成、または選択してください。

| 接続               | 用途                           | 必要なアカウント         |
| ------------------ | ------------------------------ | ------------------------ |
| Dataverse          | テーブルの読み書き             | Power Platform ユーザー  |
| Office 365 Outlook | 通知メール送信（フローで使用） | Exchange Online ユーザー |

7. 接続を選択して「**インポート**」をクリック
8. インポート完了まで待機（数分〜10分程度）

### 4.2 PAC CLI でインポート（オプション）

```powershell
# 導入先環境に認証
pac auth create --name TargetEnv --environment {target-environment-id}

# インポート
pac solution import --path ./HelpDeskGeneric_managed.zip --activate-plugins true
```

## 5. インポート後の設定

### 5.1 セキュリティロールの確認

インポート後、アプリを利用するユーザーに適切なセキュリティロールが割り当てられていることを確認してください。

1. **Power Platform 管理センター** > 環境 > セキュリティロール
2. 「**Basic User**」ロールがアプリに関連付けられていることを確認
3. 必要に応じてカスタムロールを作成・割当

### 5.2 Power Automate フローの有効化

インポート後、フローが Draft（下書き）状態になる場合があります。

1. **https://make.powerautomate.com/** にサインイン
2. 左メニューの「**ソリューション**」→「**汎用ヘルプデスク**」
3. 以下のフローを確認し、必要に応じて「**有効にする**」をクリック：

| フロー名                                 | 状態を確認    |
| ---------------------------------------- | ------------- |
| チケットが起票された場合に呼び出しされる | Active にする |
| チケットがクローズした場合にナレッジ化   | Active にする |

4. フローの接続が正しく設定されていることを確認

### 5.3 フローの接続認証

フローが「接続エラー」で失敗する場合：

1. フローを開く → 「**編集**」
2. 赤い警告が表示されている接続をクリック
3. 「**接続の修正**」で正しいアカウントを選択
4. 「**保存**」

### 5.4 アプリの動作確認

1. **https://make.powerapps.com/** > アプリ一覧
2. 「**汎用ヘルプデスク**」を起動
3. 以下を確認：
   - [ ] 左ナビに「チケット管理」「ナレッジ」「マスタデータ」の3グループが表示される
   - [ ] チケット一覧が正常に表示される
   - [ ] 「＋ 新規」でチケットを作成できる
   - [ ] チャート（📊 アイコン）が表示される
   - [ ] ナレッジ記事の一覧・作成ができる
   - [ ] カテゴリの一覧・作成ができる

## 6. 初期データの設定

### 6.1 カテゴリの登録

アプリのカテゴリは環境ごとに設定が必要です。以下は推奨カテゴリ構成です。

| 親カテゴリ       | 子カテゴリ                     |
| ---------------- | ------------------------------ |
| IT インフラ      | ネットワーク接続、セキュリティ |
| アプリケーション | Microsoft 365、業務システム    |
| アカウント管理   | パスワードリセット             |
| ハードウェア     | PC 故障・交換                  |
| その他           | —                              |

1. アプリの「カテゴリ」画面で親カテゴリを先に作成
2. 子カテゴリを作成し、「親カテゴリ」フィールドで親を選択

### 6.2 テストチケットの作成

導入先環境でテストチケットを1件作成し、以下を確認：

- チケット作成 → ステータス変更 → 解決済 → クローズのフロー
- フロー（通知・ナレッジ化）が正常に動作すること
- チャートにデータが反映されること

## 7. 運用開始後の設定（オプション）

### 7.1 メール通知の設定

フロー「チケットが起票された場合に呼び出しされる」の通知先を変更：

1. フローを編集
2. 通知先メールアドレスを導入先のヘルプデスク担当者に変更
3. 保存・有効化

### 7.2 アプリの共有

1. **make.powerapps.com** > アプリ一覧
2. 「汎用ヘルプデスク」の「**…**」>「**共有**」
3. 利用者（ユーザー/グループ）を追加
4. 適切なセキュリティロールを選択

### 7.3 Teams への埋め込み（オプション）

1. **Teams** > アプリ > 「**Power Apps**」を検索・追加
2. 「モデル駆動型アプリ」タブで「汎用ヘルプデスク」を選択
3. チーム/チャネルのタブとしてピン留め

## 8. トラブルシューティング

### インポートに失敗する

| エラー                     | 原因                     | 対処                                 |
| -------------------------- | ------------------------ | ------------------------------------ |
| Missing dependency         | 依存コンポーネントが不足 | アンマネージドでエクスポートし直す   |
| Connection reference error | 接続が未設定             | インポート時に接続を選択/作成        |
| Solution already exists    | 同名ソリューションが存在 | 既存を削除またはアップグレードを選択 |

### アプリが表示されない

- セキュリティロールが割り当てられていない → 管理者に依頼
- インポート後に公開されていない → ソリューション > 全てのカスタマイズを公開

### フローが動作しない

- 接続が切れている → フロー編集 > 接続の修正
- フローが Draft → フロー一覧から「有効にする」

### チャートが表示されない

- ブラウザキャッシュ → Ctrl+Shift+R でハードリロード
- グラフアイコンが見つからない → ビュー右上の縦3点メニュー > 「グラフの表示」
