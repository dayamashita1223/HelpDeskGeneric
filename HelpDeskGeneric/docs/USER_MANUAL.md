# 汎用ヘルプデスク — 利用マニュアル

## 1. アプリの概要

**汎用ヘルプデスク**は、社内の問い合わせ（チケット）の受付・対応・ナレッジ管理を一元化するモデル駆動型アプリです。

### できること

- チケットの起票・ステータス管理・担当者割当
- 対応履歴の記録（質問内容 → AI回答案 → 回答内容）
- ナレッジ記事の作成・レビュー・公開
- カテゴリによるチケットの分類
- チャートによる状況の可視化（7種類）
- ビジネスプロセスフロー（BPF）によるチケット・ナレッジ管理プロセスの標準化

---

## 2. アプリへのアクセス

1. ブラウザで Power Apps ポータル（https://make.powerapps.com/）を開く
2. 左メニューの「**アプリ**」をクリック
3. 「**汎用ヘルプデスク**」を選択して起動

または、以下の URL に直接アクセス：

```
https://{your-org}.crm.dynamics.com/main.aspx?appid={APP_MODULE_ID}
```

## 3. ナビゲーション

アプリの左サイドバーには3つのグループがあります。

| グループ         | メニュー     | 説明                           |
| ---------------- | ------------ | ------------------------------ |
| **チケット管理** | チケット     | 問い合わせチケットの一覧・管理 |

| **マスタデータ** | カテゴリ     | チケット・記事の分類カテゴリ   |

## 4. チケット管理

### 4.1 チケット一覧

「チケット」をクリックすると、ビュー（一覧）が表示されます。

#### ビューの切り替え

画面上部のビュー名ドロップダウンで切り替えられます。

| ビュー名                     | 内容                             |
| ---------------------------- | -------------------------------- |
| **すべてのチケット**（既定） | 全てのアクティブなチケット       |
| 全ての対応中チケット         | ステータスが「対応中」のチケット |
| 自分が対応中チケット         | 自分が担当の対応中チケット       |
| 解決済みチケット             | 解決済みのチケット               |
| Inactive チケット            | 非アクティブ化されたチケット     |

#### 表示される列

タイトル / チケット番号 / ステータス / 優先度 / 重要度 / カテゴリ / 申請者 / 担当者 / 申請日時 / 対応期限

### 4.2 チケットの新規作成

1. 一覧画面で「**＋ 新規**」ボタンをクリック
2. 以下の必須項目を入力：
   - **タイトル**: 問い合わせの件名
   - **優先度**: 高 / 中 / 低
   - **申請日時**: 問い合わせを受けた日時
   - **申請者**: 問い合わせをした人（SystemUser から選択）
3. 任意項目を入力：
   - **カテゴリ**: 分類カテゴリ
   - **重要度**: 致命的 / 重大 / 軽微 / 影響なし
   - **担当者**: 対応する担当者
   - **対応期限**: いつまでに対応するか
   - **質問内容**: 問い合わせの詳細
4. 「**保存**」をクリック
   - チケット番号が自動採番されます

### 4.3 チケットのステータス管理

チケットには以下のステータスがあります。

| ステータス           | 説明                       | 次のステータス                   |
| -------------------- | -------------------------- | -------------------------------- |
| **新規**             | 起票直後                   | 対応中 / 保留                    |
| **対応中**           | 担当者が対応中             | 解決済 / 保留 / エスカレーション |
| **保留**             | 外部回答待ち等             | 対応中                           |
| **エスカレーション** | 上位者にエスカレーション中 | 対応中 / 解決済                  |
| **解決済**           | 回答完了                   | クローズ / 対応中（再オープン）  |
| **クローズ**         | 完了                       | —                                |

### 4.4 チケットフォームの構成

チケットの詳細画面には3つのタブがあります。

**「一般」タブ**

- 基本情報: タイトル、チケット番号（自動・読取専用）、ステータス、優先度、重要度、カテゴリ
- 担当・期限: 申請者、申請日時、担当者、エスカレーション先、対応期限、解決日時、クローズ日時
- 質問内容: 問い合わせの詳細テキスト

**「対応」タブ**

- AI回答案: AI が生成した回答案（参考情報）
- 回答内容: 担当者が記入する回答
- 関連ナレッジ記事: 参考にしたナレッジ記事へのリンク

**「外部申請」タブ**

- 申請者名（外部）、申請者メール（外部）: 社外からの問い合わせ用

### 4.5 チャートで状況を可視化

チケット一覧画面の **右上のグラフアイコン（📊）** をクリックすると、チャートパネルが表示されます。

ドロップダウンで以下の7種類のチャートを切り替えられます。

| チャート名               | 種類               | 表示内容                   |
| ------------------------ | ------------------ | -------------------------- |
| ステータス別チケット件数 | 横棒グラフ         | 各ステータスの件数         |
| 優先度別チケット件数     | 棒グラフ           | 高/中/低の件数             |
| カテゴリ別チケット件数   | 横棒グラフ         | カテゴリごとの件数         |
| チケット起票推移（月別） | 折れ線グラフ       | 月ごとの起票件数トレンド   |
| チケット起票推移（日別） | 折れ線グラフ       | 日ごとの起票件数トレンド   |
| 担当者別チケット件数     | 横棒グラフ         | 担当者ごとの対応件数       |
| 担当者×ステータス別      | 積み上げ横棒グラフ | 担当者ごとのステータス内訳 |

> **ヒント**: チャートはビューのフィルタと連動します。ビューを切り替えると、チャートもそのビューのデータで描画されます。

## 5. ナレッジ記事管理

### 5.1 ナレッジ記事の作成

1. 左ナビの「**ナレッジ記事**」をクリック
2. 「**＋ 新規**」をクリック
3. 以下を入力：
   - **タイトル**: 記事のタイトル
   - **記事番号**: 管理番号（例: KB-001）
   - **記事ステータス**: 下書き → レビュー中 → 承認済み → 公開
   - **公開範囲**: 全社公開 / 部門内 / 限定公開
   - **カテゴリ**: 分類カテゴリ
   - **質問・課題**: どんな問題についての記事か
   - **回答・解決策**: 解決方法
4. 「**保存**」をクリック

### 5.2 記事ステータスの流れ

```
下書き → レビュー中 → 承認済み → 公開
                                → 非公開 → アーカイブ
```

### 5.3 ビュー

| ビュー名              | 内容                       |
| --------------------- | -------------------------- |
| Active ナレッジ記事   | 全てのアクティブ記事       |
| 公開済み ナレッジ記事 | ステータスが「公開」の記事 |
| すべての ナレッジ記事 | 全件表示                   |

## 6. カテゴリ管理

### 6.1 カテゴリの追加

1. 左ナビの「**カテゴリ**」をクリック
2. 「**＋ 新規**」をクリック
3. 以下を入力：
   - **カテゴリ名**: カテゴリの名前
   - **親カテゴリ**: 親カテゴリ（階層構造にする場合）
   - **表示順**: 一覧での並び順（数値）
   - **説明**: カテゴリの説明
4. 「**保存**」をクリック

### 6.2 カテゴリの階層構造

カテゴリは親子関係で階層化できます。

```
IT インフラ
  ├── ネットワーク接続
  └── セキュリティ
アプリケーション
  ├── Microsoft 365
  └── 業務システム
アカウント管理
  └── パスワードリセット
ハードウェア
  └── PC 故障・交換
その他
```

## 7. ビジネスプロセスフロー（BPF）

アプリには2つの BPF が含まれています。チケットやナレッジ記事のレコードを開くと、画面上部にプロセスバーが表示されます。

| BPF 名                   | 対象テーブル | 概要                         |
| ------------------------ | ------------ | ---------------------------- |
| チケット管理プロセス     | チケット     | チケットのライフサイクル管理 |
| ナレッジ記事管理プロセス | ナレッジ記事 | 記事の作成〜公開プロセス     |

---

## 8. 自動化フロー

| フロー名                                 | トリガー           | 動作                   |
| ---------------------------------------- | ------------------ | ---------------------- |
| チケットが起票された場合に呼び出しされる | チケット作成時     | 通知・自動処理         |
| チケットがクローズした場合にナレッジ化   | チケットクローズ時 | ナレッジ記事の自動生成 |

## 9. よくある質問（FAQ）

**Q. チケット番号は手入力ですか？**
A. いいえ。チケット番号フィールドは読み取り専用です。必要に応じて運用ルールに合わせて設定してください。

**Q. チャートが表示されません。**
A. チケット一覧画面の右上にあるグラフアイコン（📊）をクリックしてください。表示されない場合は Ctrl+Shift+R でブラウザキャッシュをクリアしてください。

**Q. 担当者の選択肢に自分が表示されません。**
A. 担当者フィールドは SystemUser（Dataverse ユーザー）を参照しています。Power Platform の環境にユーザーが追加されている必要があります。管理者にお問い合わせください。

**Q. カテゴリを追加したい。**
A. 左ナビの「カテゴリ」から新規作成できます。親カテゴリを指定すれば階層化も可能です。
